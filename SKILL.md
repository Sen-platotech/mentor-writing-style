# 导师文风提炼 · mentor-writing-style

把"读一批某学者的论文 → 学会他的写法 → 总结成能照着写的规范"这件事做扎实。
最终交付**一份 style-guide.md**：既描述其文风，又给出可直接套用的标题公式、摘要公式、签名结构模板、连接词库、用词频表、Do/Don't 与开工清单。

## 为什么需要这个 skill（设计动机）

学者的文风看似"只可意会"，其实高度可操作——它沉淀在**重复出现的结构、句式、连接词和措辞偏好**里。难点不在"读不懂"，而在三件容易做砸的事，本 skill 就是为绕开它们而设计：

1. **没分清谁的声音。** 中文社科论文常有：导师与学生合著、导师翻译的外文、乃至导师的导师（师承上源）的文章混在一个文件夹里。若不加区分地"平均"，学到的是一锅大杂烩，而不是目标作者本人的笔法。**作者分档是第一要务。**
2. **凭印象编造特征。** "他常说 X""他爱用排比"——若不**实际计数、不回原文逐字核对**，很容易写出语料里根本不存在的"特征"。本 skill 要求每条断言都有**可验证的出处**。
3. **只描述、不能用。** 一份"他的语言很严谨"式的赏析，写作时用不上。产出必须**操作化**：给模板、给词库、给检查清单，让人（或模型）能照着复现这种文风。

## 总体流程（六步，按顺序）

```
①作者分档 triage → ②文本抽取 → ③量化扫描(脚本) → ④质性深读 → ⑤合成 style-guide → ⑥逐字 grounding
```

逐句深读全部论文既慢又烧 token，也没必要。正确节奏是：**先靠脚本与文件名把结构、词频、作者层级"摸"出来，再把宝贵的精读用在少数最能代表本人声音的篇目上。**

---

### ① 作者分档 triage（最关键，别跳过）

先列清单，再判断每篇"在多大程度上是目标作者本人的声音"。

1. 列出文件夹中所有文档；从**文件名**与**首页作者/单位/摘要块**读出：作者（们）、年份、标题、文章类型（专著/学位论文、综述述评、理论建构、概念辨析、书评、计量分析、译作……）。
   - 用 `scripts/corpus_stats.sh` 一次性 dump 每篇的字符数、首 25 行（作者块常在此）与各级标题，省去逐个打开。
2. 按"本人声音"分四档（**权重从高到低**）：
   - **独著 / 第一作者**：最高权重，语言与结构都全力模仿。
   - **与晚辈（学生）合著**：**取结构与套路、降低语言权重**——遣词可能出自学生之手。
   - **本人翻译的外文**：**只取术语译名**（反映其译词选择），**不取论证结构**——那是原作者的思路。
   - **他人 / 师承上源**（如导师的导师、被引作者）：单列或排除，不作语言范本。
3. **时间加权**：作者高产、亲自操刀的"黄金时段"内的篇目权重更高；越晚近、越可能是学生代笔的合著，权重越低。**主动问用户**这个时段与人物关系——文件名里的人名谁是师、谁是生、哪几年是本人巅峰，用户最清楚，模型猜不准。
4. **把分档结果做成一张表给用户确认**，再进入精读。这一步的确认能避免后面把功夫下错人身上。

> 中文社科特有的作者关系、译作识别等常识，见 `references/chinese-academic-cues.md` 的"作者分档"节。

### ② 文本抽取

- 用脚本批量把 PDF/DOCX 抽成 .txt（`pdftotext` 为主，`markitdown` 兜底）。
- **识别扫描件**：字符数接近 0 或远低于同类的，多半是图片型 PDF。需要 OCR，成本高——**若本人声音的文本篇目已足够，可跳过扫描件并注明**，不影响结论。

### ③ 量化扫描（能脚本化的都脚本化）

用 `scripts/corpus_stats.sh` 跨全部（或高权重）篇目提取：
- **各级标题**（`一、`/`（一）`/`1.`）→ 找出**反复出现的文章骨架与"签名结构模板"**（多篇同构的章节安排，是最值钱的发现）。
- **标题与摘要句式**的共性。
- **短语/连接词频次**：骨架词、过渡词、并列短语——**用数字说话**，别凭感觉。脚本内置一份中文社科常见清单，可按需替换。

### ④ 质性深读（只读高权重篇目的关键段）

挑 3–6 篇最高权重篇目，**各读引言、一个正文小节、结语**即可，重点抓：
- **开篇修辞动作**：怎么起（时间/学科定位？）、怎么用"但/然而"转折、怎么论证"本文很有必要"（常见套路：先肯定成就 → 转折 → 排比点出研究缺口 → 引出本文）。
- **段落展开**：主题句 → `首先/其次/再次` 或 `其一/其二/其三` 排比分述 → "由此可推断"式小结。
- **下定义的方式、对举与让步、判断的分寸**（强断言还是多用"往往/一定程度上"）。
- **句子节奏与长度**。
> 各项在中文社科里的典型样态与识别要点，见 `references/chinese-academic-cues.md`。

### ⑤ 合成 style-guide

严格按 `references/style-guide-template.md` 的结构产出，做到**描述 + 可操作并重**：一段话文风画像、标题/骨架/摘要公式、**签名结构模板的逐小节拆解**、段落与论证肌理 + 连接词库、用词与术语（带实测频次）、引证体例、Do/Don't、开工清单，文末附**作者分档表**。
- 模板要**拆到小节粒度**并给"填空式套用示意"——这才是能直接驱动写作的部分。
- 文件名默认 `<作者>文风_style-guide.md`，存到用户的工作文件夹。
- 参考 `examples/` 里的演示示例感受目标质量与密度（该示例为虚构，仅示范输出形态）。

### ⑥ 逐字 grounding（交付前必做）

- style-guide 中**每一条特征断言，都要配一句来自语料的原文引文**，并标注【篇名·年份】。
- 用 `grep`（去换行后匹配）回原文**逐字核对**引文无误；核对频次声明。
- 凡无法在语料中坐实的判断，**删除或显式标注"存疑"**。绝不编造。这是这份指南可信的根本。

## 与用户协作的要点

- **先确认目标作者与分档表，再精读**；动笔前把"本次要分析谁、读哪些篇、什么时段权重最高"对齐。
- 节奏上可**分阶段停下确认**（先给分类表，再给指南），尤其当语料多、关系复杂时。
- 保持**简洁直接**，少铺陈。

## 不要做的事

- 不要把合著、译作、他人文章与目标作者独著**一视同仁地平均**。
- 不要**凭印象**断言"他常用 X"——没计数、没出处就不写。
- 不要只产出**赏析式**描述而无可套用的模板与清单。
- 不要为省事**逐篇全文精读**——先脚本摸底，精读只给高权重篇目的关键段。
- 不要硬上 OCR 啃扫描件，除非本人声音的文本篇目不足。

## 资源

- `scripts/corpus_stats.sh` — 批量抽取文本、报字符数（识别扫描件）、dump 作者块、列各级标题、按清单统计词频。用法见脚本头注释。
- `references/chinese-academic-cues.md` — 中文人文社科论文的标题/摘要/定义/词对/连接词/引证体例的典型样态，以及作者分档常识。深读与合成时读它。
- `references/style-guide-template.md` — 最终 style-guide 的固定结构（照此产出）。
- `examples/` — 一份**虚构的**演示示例，体会目标输出的结构与密度。
